## 经验教训

### 2026-03-10：网络请求工具的可靠性

- 问题：`fetch_url` 工具和 `terminal curl` 在当前环境下多次超时或失败
- 有效方案：`python_repl` + `requests` 库是最可靠的网络请求方式
- 下次提示：遇到网络请求任务时，优先使用 `python_repl` + `requests`，设置 timeout 参数（10-15秒）

### 2026-03-10：天气查询 API 选择

- 问题：wttr.in 接口不稳定，多次超时
- 有效方案：Open-Meteo API 响应更快、更稳定
- 下次提示：天气查询优先使用 Open-Meteo API（需提供经纬度坐标）

### 2026-03-10：知识库 Excel 检索方法

- 问题：如何高效检索 Excel 文件中的结构化数据
- 有效方案：使用 `python_repl` + `pandas` 进行数据分析
- 关键步骤：
  1. 用 `pd.read_excel()` 读取文件
  2. 用 `df[df['列名'] == '条件']` 进行筛选
  3. 用 `df.groupby()` 进行分组统计
  4. 用 `df.sort_values()` 进行排序
  5. 用 `df.agg()` 进行聚合计算（mean, max, min, sum）
- 示例场景：查询某部门人数、工资最高者、各部门对比等
- 下次提示：涉及 Excel 数据查询时，优先用 pandas 进行结构化分析

### 2026-03-10：知识库 PDF 检索方法

- 问题：如何从 PDF 中提取特定信息（如股东信息）
- 有效方案：使用 `python_repl` + `pdfplumber` 进行 PDF 文本提取
- 关键步骤：
  1. 用 `pdfplumber.open()` 打开 PDF
  2. 遍历 `pdf.pages` 获取各页内容
  3. 用 `page.extract_text()` 提取文本
  4. 用字符串匹配或正则表达式定位目标信息
- 示例场景：从财报中提取股东持股信息
- 下次提示：涉及 PDF 信息提取时，优先用 pdfplumber，注意定位关键词

### 2026-03-10：多轮查询的上下文记忆

- 问题：用户连续提问相关问题时，如何保持上下文一致性
- 有效方案：
  1. 在会话中记住之前的查询结果
  2. 后续问题可复用之前的数据，无需重新查询
  3. 用户追问时直接回答，体现"记忆"能力
- 示例场景：先查销售部门，再查技术部门，最后问"其他部门"，用户问"你还记得销售部门最高工资吗"
- 下次提示：多轮对话中保持上下文，避免重复查询，提升响应效率

### 2026-03-10：数据分析结果的可视化呈现

- 问题：如何让数据查询结果更易读
- 有效方案：
  1. 使用 Markdown 表格呈现结构化数据
  2. 关键数据加粗突出
  3. 提供 Top N 排行榜
  4. 提供统计汇总（平均、最高、最低、总和）
  5. 多维度对比时提供对比表格
- 示例场景：部门工资分析、股东排名等
- 下次提示：数据结果用表格+排行榜+汇总的形式呈现，提升可读性
